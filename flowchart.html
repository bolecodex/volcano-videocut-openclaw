Shots → AI Video 执行流程

shots-to-ai-video Skill  |  Seedance Lite 三模式自动选择

步骤 1
📂 读取资产清单
解析项目数据文件，构建角色图片映射表。
shots/_manifest.yaml *_角色资产.md
  • style_base → 全局风格基准
  • characters → 角色 ID 列表
  • 角色图片映射 → @id → image_url
步骤 2
🎬 选择场景
展示场景列表（含镜头数、图片/音频完整性），用户选择目标场景或 "all"。
读取选中场景的分镜 YAML 文件。
步骤 3
🔄 遍历每个镜头 (shot)
对场景中的每个 shot，依次执行模式判断 → 构建提示词 → 调用 API → 获取结果。
角色有参考图？
✅ 有角色参考图 P1 最优
🎯
参考图生视频
reference-to-video
传入 1-4 张角色参考图
提示词：动作 + 场景
角色一致性最好
❌ 无参考图 / 有分镜图 P2
🖼️
图生视频
image-to-video
分镜图作为首帧
提示词：动作变化描述
画面延续性好
❌ 无任何图片 P3
📝
文生视频
text-to-video
纯文本提示词
包含 style_base + 角色外貌
灵活度最高
📐 提示词构建策略（按模式差异化）
模式 风格词 (style_base) 角色外貌 动作/场景 镜头运动
🎯 reference 不需要 不需要 需要 需要
🖼️ image 不需要 不需要 需要 需要
📝 text 需要 需要 需要 需要
API
🚀 提交任务 & 轮询结果
调用 submit_task 提交视频生成任务，记录 task_id。
  • 提交后等待 30 秒
  • 每 15 秒 调用 get_task 查询
  • 超时 5 分钟 标记失败
  • 失败自动重试 1 次
成功获取 video_url
步骤 4
✅ 更新 YAML & 展示结果
每个镜头生成成功后，立即写入 YAML 并展示。
video_url video_status video_mode video_duration video_generated_at
  • 写入分镜 YAML 的 5 个新字段
  • 展示视频链接、模式、时长
  • 标记 ✅ / ❌ 状态
还有下一个镜头？
🔁 返回「遍历下一个镜头」直到场景所有镜头处理完毕
🎉 场景视频生成完毕
