# 阶段 2：分镜脚本（逐帧确认）

目标：把剧本拆成可执行的分镜列表，并保存到 `.yiwa/<project_id>/storyboards.json`（以及写回到 `script.json` 的 `storyboards` 字段）。

## 1. 获取分镜生成提示词

默认从 `.yiwa/<project_id>/script.json` 读取剧本：

```bash
printf '%s' '{"project_id":"..."}' | node "scripts/expand-storydoard-script.js"
```

输出：

```json
{ "prompt": "..." }
```

## 2. 先产出“分镜关键信息”给用户确认

建议先生成简版（不展开超长细节），并展示给用户：

- 分镜总数与总时长是否合理
- 每帧关键信息：shot_number / storyboard_id（如有）、场景、角色、道具、画面要点、台词、时长

然后停下来问用户是否需要调整整体节奏/内容（快慢、删减、补镜头、重排等）。

## 3. 再逐帧生成“详细分镜”

按照以下流程默认一次只生成 1 个分镜的详细描述（如果用户要求全部生成，则一次生成所有分镜）：

1. 生成该帧的详细分镜 JSON（只输出 JSON）
2. 展示给用户分镜信息，并询问是否修改该分镜
3. 用户确认后立刻保存该分镜

保存命令（`storyboard_json` 是一个数组，即使只保存 1 帧也要用数组包起来）：

```bash
printf '%s\n' '{"project_id":"...","storyboard_json":"[ { ...第i帧分镜JSON... } ]"}' | node "scripts/save-storyboards.js"
```

说明：

- `storyboard_json` 可以直接传数组对象，也可以传 JSON 字符串；推荐传 JSON 字符串，避免换行导致解析失败。
- `printf` 建议用 `%s\n` 结尾补一个换行，避免终端把提示符贴在同一行，看起来像在末尾追加了 `~` 等多余字符。
- 若未提供 `storyboard_id`，脚本会根据 `project_id + shot_number` 自动生成稳定 ID。

## 4. 完成后检查与用户确认

完成全部帧后：

- 告知保存位置：`.yiwa/<project_id>/storyboards.json`
- 让用户确认是否需要整体回看并修改某几帧

如需回看当前所有角色/场景/分镜列表，可用：

```bash
printf '%s' '{"project_id":"..."}' | node "scripts/list-entities.js"
```
