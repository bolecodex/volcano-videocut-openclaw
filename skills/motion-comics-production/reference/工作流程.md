# 工作流程总览

## 目录与产物

所有脚本默认把状态与产物写在当前工作目录下的 `.yiwa/`：

```
.yiwa/<project_id>/
  script.json                # 剧本（完整 JSON）
  characters.json            # 角色（含 image_url/voice）
  props.json                 # 道具（含 image_url）
  scenes.json                # 场景（含 image_url）
  storyboards.json           # 分镜列表（含 image_url/video_url/local_path/audio_path 等）
  images/                    # 下载的分镜图（sb_*.jpeg）
  videos/                    # 下载的视频片段（<storyboard_id>.mp4）
  audio/                     # 生成的配音（<storyboard_id>.mp3）
  merged/                    # 合成阶段的中间文件（逐段合并音轨后的 mp4）
  output/
    <project_id>.srt         # 字幕（可选）
    <project_id>.mp4         # 最终成片
```

## 阶段拆分

### 阶段 0：安装与环境

先完成依赖与 `.env` 配置：见 [安装与环境配置](./安装与环境配置.md)。

### 阶段 1：剧本与资产（project_id 起点）

- `scripts/expand-script.js`：把「故事想法」扩写为剧本生成提示词，并生成 `project_id`
- 你根据提示词生成“严格 JSON”剧本
- `scripts/save-script-info.js`：保存 `script.json`，同时抽取并写入角色/道具/场景数据

详细操作见 [阶段1-剧本与资产](./阶段1-剧本与资产.md)。

### 阶段 2：分镜脚本（逐帧确认）

- `scripts/expand-storydoard-script.js`：生成分镜提示词（读取 `.yiwa/<project_id>/script.json`）
- 你根据提示词生成分镜 JSON（推荐先出“关键信息”，再逐帧展开）
- `scripts/save-storyboards.js`：增量保存分镜

详细操作见 [阶段2-分镜脚本](./阶段2-分镜脚本.md)。

### 阶段 3：素材图生成（角色/道具/场景）

- `scripts/generate-image.js`：生成并回填素材图 URL

详细操作见 [阶段3-素材图生成](./阶段3-素材图生成.md)。

### 阶段 4：分镜图生成（引用素材图）

- `scripts/generate-storyboard.js`：基于分镜描述生成分镜图（会自动参考已生成的角色/道具/场景图）
- `scripts/modify-storyboard.js`：修改某帧描述并清空该帧图/视频，便于重做

详细操作见 [阶段4-分镜图生成](./阶段4-分镜图生成.md)。

### 阶段 5：视频片段生成

- `scripts/generate-video.js`：把分镜图转成视频片段并下载到本地

详细操作见 [阶段5-视频片段生成](./阶段5-视频片段生成.md)。


### 阶段 6：合成输出

- `scripts/compose-video.js`：拼接全部片段；若存在音频则先逐段合并音轨；若存在 SRT 则烧录字幕

详细操作见 [阶段6-合成输出](./阶段6-合成输出.md)。

## 交互原则（必须遵守）

- 每个阶段结束都要展示关键结果并等待用户确认（继续/修改/重做），不要自动跳到下一阶段。
- 需要模型生成 JSON 的地方必须只输出 JSON（不带解释文字），否则脚本无法保存或后续解析失败。
