# 阶段 3：素材图生成（角色/道具/场景）

目标：为角色、道具、场景生成参考图，并回填到 `.yiwa/<project_id>/characters.json` / `props.json` / `scenes.json`。

## 0. 准备：拿到要生成的 ID 列表

```bash
printf '%s' '{"project_id":"..."}' | node "scripts/list-entities.js"
```

根据输出里的 `characters.list` / `props.list` / `scenes.list` 拿到每个实体的 `id`。

## 1. 生成角色图

```bash
printf '%s' '{"type":"char","id":"角色ID","project_id":"..."}' | node "scripts/generate-image.js"
```

输出：

```json
{ "id": "...", "name": "...", "image_url": "..." }
```

展示给用户（必须用图片语法）：

```md
![角色名](image_url)
```
同时把图片保存的本地路径告诉用户。

## 2. 生成道具图

```bash
printf '%s' '{"type":"prop","id":"道具ID","project_id":"..."}' | node "scripts/generate-image.js"
```

## 4. 交互与重做

- 每张图生成后都要展示并询问是否满意；用户要求重做时，直接对同一 ID 再跑一次即可（`image_url` 会被覆盖更新）。
- 全部素材图完成后，停下来让用户确认“是否继续生成分镜图”。
