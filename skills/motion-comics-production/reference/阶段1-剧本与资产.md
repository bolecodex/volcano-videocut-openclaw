# 阶段 1：剧本与资产（角色/场景/道具）

目标：从“故事想法”得到可保存的 `script.json`，并抽取角色/场景/道具清单。

## 1. 生成剧本提示词与 project_id

```bash
printf '%s' '{"idea":"用户的故事想法"}' | node "scripts/expand-script.js"
```

输出：

```json
{ "project_id": "...", "prompt": "..." }
```

记录 `project_id`，后续所有步骤都要用。

## 2. 根据 prompt 生成“严格 JSON”剧本

- 只输出 JSON，不要其他文字（否则无法保存/解析）。
- JSON 结构必须严格遵循 prompt 的要求。

## 3. 保存剧本并抽取资产

把上一步生成的 JSON 串作为 `script_json` 传入：

```bash
printf '%s' '{"project_id":"...","script_json":"{...}"}' | node "scripts/save-script-info.js"
```

说明：

- `script_json` 必须是 JSON 字符串（推荐在对话里对 JSON 做 `JSON.stringify` 后再粘贴）。
- 保存位置：`.yiwa/<project_id>/script.json`

脚本会返回抽取后的资产清单（角色/道具/场景），同时写入 `.yiwa/<project_id>/characters.json`、`props.json`、`scenes.json`。

## 4. 给用户展示关键信息并等待确认

总结以下关键信息展示给用户，注意不要直接把原始 JSON 展示给用户：

- 剧名、推荐画风/视觉风格（如果剧本里有）
- 每集标题与梗概（如有分集）
- 角色列表（名字、性格/描述、可选配音 voice）
- 道具列表（名称、外观/象征/描述）
- 场景列表（名字、描述）
- 产物位置(存放原始JSON的路径)

然后停下来询问用户是否修改：

- 修改剧情/角色/场景/道具描述
- 修改画风（例如“改成写实风 / 水彩风 / 中国水墨风”等）
- 或直接说“继续”进入分镜

## 5. 修改方式（两种）

### A) 直接在模型侧重写 script_json

按用户要求修改后，重新生成“严格 JSON”并重复第 3 步保存。

### B) 单独更新项目画风（只改风格，不改剧本）

当用户只想改画风时，用 `set-style.js` 更新项目风格：

```bash
printf '%s' '{"project_id":"...","style":"realistic"}' | node "scripts/set-style.js"
```

可选自定义完整风格提示词：

```bash
printf '%s' '{"project_id":"...","style":"custom","style_prompt":"你希望的具体画风描述"}' | node "scripts/set-style.js"
```
