# 阶段 4：分镜图生成

目标：为每个分镜生成 16:9 分镜图，并保存到本地，同时回填 `storyboards.json` 的 `image_url`。

## 0. 准备：拿到 storyboard_id 列表

```bash
printf '%s' '{"project_id":"..."}' | node "scripts/list-entities.js"
```

根据输出里的 `storyboards.list` 获取 `id`（即 `storyboard_id`）。

## 1. 逐帧生成分镜图

```bash
printf '%s' '{"storyboard_id":"...", "project_id":"..."}' | node "scripts/generate-storyboard.js"
```

输出：

```json
{ "storyboard_id": "...", "image_url": "...", "style_prompt": "..." }
```

脚本会把分镜图下载到：

- `.yiwa/<project_id>/images/`

展示给用户（必须用图片语法）：

```md
![分镜 storyboard_id](image_url)
```
同时把图片保存的本地路径告诉用户。

每帧生成后停下来问用户是否修改该帧（构图/动作/镜头/表情/道具/场景等）。

## 2. 修改某帧并重做

当用户要改某帧描述时：

```bash
printf '%s' '{"storyboard_id":"...","new_description":"新的描述"}' | node "scripts/modify-storyboard.js"
```

然后重新生成该帧：

```bash
printf '%s' '{"storyboard_id":"...","project_id":"..."}' | node "scripts/generate-storyboard.js"
```

## 3. 敏感内容与失败处理

如果生成失败或触发敏感拦截：

- 优先让用户把描述改得更“安全”（无血腥暴力、无武器、无政治标识、不要出现可读文字/Logo/水印等）
- 再重跑该帧的 `generate-storyboard.js`
