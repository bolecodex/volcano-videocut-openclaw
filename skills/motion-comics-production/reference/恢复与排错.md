# 恢复与排错

## 1. 恢复中断的项目

当用户提供 `project_id` 时，先查看项目进度：

```bash
printf '%s' '{"project_id":"..."}' | node "scripts/status.js"
```

输出字段 `stage` 含义：

- `script`：尚未保存剧本或找不到 `script.json`
- `entities_extracted`：已保存剧本并抽取出角色/场景（可进入素材图生成）
- `images_generating`：素材图开始生成但未完成
- `all_images_done`：素材图齐全（可进入分镜图生成）
- `videos_generating`：视频片段开始生成但未完成
- `ready_to_compose`：视频片段齐全（可进入合成）

按 stage 继续对应阶段：

- `script` → [阶段1-剧本与资产](./阶段1-剧本与资产.md)
- `entities_extracted` / `images_generating` → [阶段3-素材图生成](./阶段3-素材图生成.md)
- `all_images_done` → [阶段4-分镜图生成](./阶段4-分镜图生成.md)
- `videos_generating` / `ready_to_compose` → [阶段5-视频片段生成](./阶段5-视频片段生成.md) / [阶段7-合成输出](./阶段7-合成输出.md)

## 2. 常见报错与处理

### A) ARK_API_KEY 未设置 / 401

- 现象：提示 `ARK_API_KEY 未设置` 或请求返回 401
- 处理：按 [安装与环境配置](./安装与环境配置.md) 配置并验证 `ARK_API_KEY`

### B) ffmpeg / ffprobe not found

- 现象：`compose-video.js` / `generate-subtitle.js` 报错找不到 ffmpeg/ffprobe
- 处理：安装 ffmpeg，并保证 `ffmpeg`/`ffprobe` 在 PATH（macOS 常用 `brew install ffmpeg`）

### C) Storyboard has no image yet

- 现象：跑 `generate-video.js` 报错该帧没有 image
- 处理：先跑 [阶段4-分镜图生成](./阶段4-分镜图生成.md) 为该帧生成分镜图

### D) script.json not found

- 现象：跑 `expand-storydoard-script.js` 报错找不到 `.yiwa/<project_id>/script.json`
- 处理：确认命令在 `motion-comics-production/` 目录执行；再回到 [阶段1-剧本与资产](./阶段1-剧本与资产.md) 重新保存

### E) 生成敏感拦截 / 画面不合规

- 现象：生成图/视频失败，提示敏感内容检测
- 处理：把该帧描述改得更安全（避免血腥暴力、武器、政治标识、可读文字/Logo/水印），然后重跑对应生成命令
