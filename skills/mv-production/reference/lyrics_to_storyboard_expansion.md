# 歌词驱动的分镜扩写规则（宿主 Agent 参考）

> 本文件为宿主 Agent 提供从歌词到分镜扩写的指导，`mv-production` 仅消费已生成的 `storyboard.json`。

## 1. 输入

- 纯歌词文本：`output_dir/lyrics.txt`；
- 纠错后 ASR 结果：`output_dir/asr_corrected.json`；
- 前奏信息：`output_dir/timing_meta.json`（含 `intro_ms`）；
- 可选：`song_meta.json`（角色、情绪、场景等标签）。

## 2. 目标

在不改变歌词实际时间轴的前提下，对每一句/每一段歌词进行适度的「故事化扩写」，生成具有视觉冲击力且可执行的分镜：

- 为 intro 段（前奏）设计氛围镜头；
- 为每行歌词匹配一个或多个镜头；
- 保证镜头时间范围与 `asr_corrected` 中的时间一致或接近；
- 将扩写的内容写入分镜字段：
  - `visual_content`
  - `image_description`
  - `video_description`

## 3. 基本原则

1. **时间优先**  
   - 每条歌词的时间窗由 `asr_corrected.json` 中的 `start_time_ms` / `end_time_ms` 决定；
   - 扩写后的镜头时长总和应与音频时长相符，允许轻微浮动；
   - intro 段（`0 ~ intro_ms`）的镜头不得出现明显口型。

2. **歌词含义 → 画面隐喻**  
   - 优先使用歌词中的意象（地点、道具、人物行为）作为画面元素；
   - 可以加入符号化隐喻（如 KPI → 紧箍咒、通勤 → 取经路等），但画面风格要与歌曲整体一致。

3. **情绪曲线**  
   - 识别 Verse / Chorus / Bridge 的情绪变化，在画面中通过构图、色调、运镜体现；
   - 例如：
     - Verse：更多叙事和环境铺垫；
     - Chorus：情绪爆发、构图更饱满、运镜更激烈。

## 4. 与 ASR 的配合

- 对于每一行歌词：
  - 使用其时间窗中心作为主要镜头的时间锚点；
  - 若需要拆为多个镜头：
    - 在同一时间窗内划分多个子片段；
    - `dialogue` 字段只在关键镜头中填入歌词，其余镜头 `dialogue` 可为空或复用同一句；
- intro 段：
  - 使用 `intro_ms` 前的时间生成无口型画面（如环境、城市夜景、角色背影等）。

## 5. 录音棚等「口型可见」场景的额外约定

当用户场景为**录音棚演出**、**舞台演唱**、**对嘴 MV** 等需要口型与歌词严格对齐时，分镜需**增加时间节点并标明当前演唱歌词**，便于视频生成与字幕一致：

- **时间节点**：每个镜头的 `start_time` / `end_time`（或 `start_time_ms` / `end_time_ms`）必须与 `asr_corrected.json` 中对应歌词行的时间窗一致或为其子区间。
- **当前歌词**：
  - 在镜头对象中显式写出该时间段内人物正在演唱的歌词，例如：
    - `dialogue`：该镜头对应的一句或半句歌词（与口型一致）；
    - 可选 `current_lyric`：与 `dialogue` 一致或为整句，便于下游展示/调试；
  - 若为 intro 段（0 ~ 首句歌词开始），则 `dialogue` 应为空或标注为「前奏/准备」，画面描述为**准备状态**（如闭口、拿麦、看提词），不安排歌唱口型。
- **示例**：若 asr_corrected 显示 1s–2s 有歌词「唐僧说这个月 KPI 还没达标」：
  - 0–1s 镜头：`dialogue` 为空或「前奏」，`video_description` 写「人物在录音棚准备状态，未开口」；
  - 1–2s 镜头：`dialogue` 为「唐僧说这个月 KPI 还没达标」（或按切分填对应子句），`video_description` 写「人物歌唱状态，演唱该句」。
- 宿主 Agent 在生成 `storyboard.json` 时，应根据 `asr_corrected.json` 的每行 `lyric` 与 `start_time_ms`/`end_time_ms` 为每个镜头填写上述字段，保证时间节点与当前歌词一一对应。

## 6. 按场景区分 A-roll 与 B-roll

分镜设计需结合**场景类型**区分主镜头（A-roll）与补充镜头（B-roll），便于叙事主线和节奏控制。

### A-roll（A 轨 / 主镜头）

- **含义**：承载「主要内容」的核心画面，推进叙事与信息。
- **典型用法**：
  - 主持人/采访对象对着镜头说话；
  - 访谈正反打、Vlog 自述、课堂讲解；
  - 在 MV/歌曲场景下：**角色对着镜头演唱、对嘴、独白**，即「正在唱这句歌词」的画面。
- **分镜约定**：
  - 有明确**口型/歌词**的镜头应标为 A-roll；
  - `dialogue` 有内容、且时间窗与 asr_corrected 某行歌词对应的镜头，一般为 A-roll；
  - 用于承载「谁在唱、唱什么」的主线。

### B-roll（B 轨 / 补充镜头 / 覆盖镜头）

- **含义**：配合主线、补画面、提升氛围的辅助素材。
- **典型用法**：
  - 环境空镜、细节特写、过程镜头（如手部操作、乐器特写）；
  - 街景、产品细节、现场氛围、观众反应；
  - 在 MV/歌曲场景下：**不直接对嘴的**环境、道具、肢体细节、转场过渡等。
- **分镜约定**：
  - 无口型或仅作氛围/过渡的镜头应标为 B-roll；
  - 可用于**覆盖剪辑点**（例如 A-roll 剪掉停顿/口误时用 B-roll 盖住跳切）；
  - 增强信息表达（歌词提到某意象时切到对应 B-roll）；
  - 调节节奏，避免全程单一主镜头。

### 在 storyboard 中的落地

- 每个镜头对象应包含 **`roll_type`**（或 `shot_role`）：`"a_roll"` 或 `"b_roll"`。
- **A-roll**：`roll_type: "a_roll"`，通常对应有 `dialogue`、与歌词时间窗对齐、人物正面/对嘴演唱。
- **B-roll**：`roll_type: "b_roll"`，通常 `dialogue` 为空或仅氛围，画面为环境/细节/过程/过渡。
- 示例（采访/叙述类）：
  - A-roll：受访者说「我们用三个月完成了改版」→ 对镜头说话的主镜头；
  - B-roll：团队开会、敲代码、产品界面、办公室、用户使用场景 → 补充画面与节奏。

宿主 Agent 在生成分镜时，应根据歌词结构、节奏与场景类型，合理穿插 A-roll 与 B-roll，并在输出 JSON 中为每个镜头填写 `roll_type`。

## 7. 输出约定

扩写完成后，宿主 Agent 应生成 `output_dir/storyboard.json`：

- 结构可参考 `reference/storyboard_prompt_spec.md` 中的字段定义；
- 需要确保：
  - 每个镜头包含 `shot_id`、`duration`（或可推导时长）、`image_description`、`video_description` 等；
  - 引用角色三视图与场景参考图时，使用相对路径（如 `character_views/front.png`）。
