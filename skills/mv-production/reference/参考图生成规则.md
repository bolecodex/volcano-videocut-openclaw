# 参考图生成规则（角色 / 场景 / 道具）

本规则约定 M4 阶段参考图的生成方式。**M4 在 M5 分镜之后执行**，参考图须**基于分镜**生成：角色三视图、场景图、道具图均应根据 **storyboard.json** 中出现的角色、场景、道具（及宿主从中提炼的资产列表）生成，保证与分镜一致。

## 1. 生成范围

| 类型 | 说明 | 输出目录 | 与剧本关系 |
|------|------|----------|------------|
| **角色三视图** | 正面 / 侧面 / 背面，保证人物形象一致 | `output_dir/character_views/front.png` 等 | 根据剧本/设定中的角色描述生成 |
| **场景图** | 剧本中出现的核心场景 | `output_dir/key_scenes/{scene_id}.png` | 根据剧本中的 **scene** 列表生成，每个 scene_id 对应一张图 |
| **道具图** | 剧本中出现的关键道具 | `output_dir/props/{prop_id}.png` | 根据剧本中的 **props** 列表生成，每个 prop_id 对应一张图 |

## 2. 资产配置格式（供脚本与宿主 Agent 使用）

宿主 Agent 在 M4 阶段，应**从 storyboard.json 提取**分镜中出现的「角色 + 场景 + 道具」清单（可结合 lyrics、`song_meta.json` 补充描述），并写成 JSON 配置文件，供 `scripts/generate_reference_image.py` 使用。

**推荐结构（`assets.json` 或由宿主 Agent 写入的 `output_dir/reference_assets.json`）：**

```json
{
  "character": {
    "description": "西游记蝎子精，现代职场装，头戴监听耳机，神情略带疲惫与自嘲，2D 卡通偏写实风格"
  },
  "scenes": [
    {
      "id": "scene_studio",
      "description": "专业录音棚内景，复古麦克风与调音台，暖色灯光，氛围感强"
    },
    {
      "id": "scene_office",
      "description": "现代办公室工位，电脑与文件，窗外城市夜景"
    }
  ],
  "props": [
    {
      "id": "prop_mic",
      "description": "复古金属麦克风，特写，产品级质感"
    },
    {
      "id": "prop_headphone",
      "description": "专业监听耳机，黑色，侧面特写"
    }
  ]
}
```

- **character**：单个对象，至少包含 `description`（或 `prompt`/`name`），用于生成角色三视图的共用描述。
- **scenes**：数组，每项至少包含 `id`（对应分镜中的 `scene` 字段）和 `description`（或 `prompt`/`name`），用于生成场景图。
- **props**：数组，每项至少包含 `id`（对应分镜中的 `props` 数组元素）和 `description`（或 `prompt`/`name`），用于生成道具图。

**一致性要求**：本配置中的 `scenes[].id`、`props[].id` 须与 storyboard.json 中的 `scene`、`props` 一致（因配置即由分镜提取），以便在视频生成阶段正确引用对应参考图。

## 3. 脚本调用方式

在 `mv-production` 根目录或 `scripts` 所在目录执行：

```bash
# 根据配置文件生成全部参考图（角色三视图 + 场景图 + 道具图）
uv run python scripts/generate_reference_image.py \
  --config output_dir/reference_assets.json \
  --output-dir output_dir \
  --mode all

# 仅生成场景图
uv run python scripts/generate_reference_image.py \
  --config output_dir/reference_assets.json \
  --output-dir output_dir \
  --mode scenes

# 仅生成道具图
uv run python scripts/generate_reference_image.py \
  --config output_dir/reference_assets.json \
  --output-dir output_dir \
  --mode props
```

**支持参考图（对应文生图接口的 `image` 参数，见 `文生图.md`）**：

- 命令行：`--reference-image <路径或 URL>` 可多次传入，最多 14 张；
- 配置文件：在 config JSON 中增加 `reference_images` 或 `ref_images`（字符串或字符串数组），为用户提供的参考图路径/URL。

生成时会将参考图传给接口，实现「单图生图/多图生图」，保证风格与用户参考一致。示例：

```bash
# 基于用户参考图生成全部参考图（角色三视图、场景图、道具图）
uv run python scripts/generate_reference_image.py \
  --config output_dir/reference_assets.json \
  --output-dir output_dir \
  --mode all \
  --reference-image output_dir/ref_images/用户图.png
```

生成完成后，脚本会在 `output_dir` 下写入：

- `character_views/front.png`、`side.png`、`back.png`
- `key_scenes/{scene_id}.png`
- `props/{prop_id}.png`
- `reference_images_index.json`（路径索引，便于分镜与视频生成引用）

## 4. 与分镜的衔接

- 分镜 JSON 中应通过字段引用上述参考图，例如：
  - `character_reference`: `{ "front": "character_views/front.png", "side": "character_views/side.png", "back": "character_views/back.png" }`
  - `scene_reference`: 按 scene_id 映射，如 `{ "scene_studio": "key_scenes/scene_studio.png" }` 或数组 `["key_scenes/scene_studio.png"]`
  - `props_reference`: 按 prop_id 映射，如 `{ "prop_mic": "props/prop_mic.png" }`
- 视频生成（Seedance 等）时，可根据分镜的 `scene`、`props` 从上述路径加载对应参考图，保证画面与剧本一致。

## 5. 注意事项

- 场景图、道具图的 **description** 应尽量与分镜中 `image_description` / `visual_content` 对场景、道具的表述一致或互补，避免风格分裂。
- 若用户已提供参考图并选择「直接使用」，可将用户图放入 `ref_images/`；若用户选择「基于参考图再生成」，则**必须**以用户图为参考完整生成角色三视图、场景图、道具图，不得跳过。
- 用户参考图（`ref_images/`）仅作风格/生图参考，**不得**默认作为视频首帧；首帧仅来自上一镜尾帧或分镜中显式指定的首帧图。
- **风格以哪个为准**（强约定）：
  - 若提供了用户参考图（`--reference-image` / config `reference_images`），则**以用户参考图为风格基准**，三类参考图均应使用同一组参考图作为 `image` 输入，保证风格收敛。
  - 若未提供用户参考图，则默认启用脚本的 **auto-style-anchor**：先生成 `character_views/front.png`，再把它作为后续场景图/道具图的参考图输入（同样走 `image` 参数），以角色正面作为风格锚点；避免场景/道具各自跑偏。
