# 敏感词音频处理 - Python 脚本方案（Review）

## 目标

将 `敏感词音频处理.md` 第 1–4 步写成可复用的 Python 脚本，供后续流程直接调用。

## 流程对应关系

| 文档步骤 | 说明                                          | 脚本实现                                                                                                                                   |
| -------- | --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| 1        | 读取敏感词列表中的时间间隔 `sensitivity.json` | `load_intervals()` 解析 JSON，收集所有 `start_time` / `end_time`（毫秒）                                                                   |
| 2        | 使用 ffmpeg，替换文件为 `asserts/dd6a0e.mp3`  | 脚本默认使用 `审核/asserts/dd6a0e.mp3`，可通过 `--beep` 覆盖                                                                               |
| 3        | 屏蔽音 1017ms：超过则剪切，少于则拼接后替换   | **超过 1017ms**：对 beep 做 `-stream_loop` 循环后 `atrim` 到区间长度；**少于 1017ms**：对 beep 做 `atrim` 到区间长度（单段截取，无需拼接） |
| 4        | 核心逻辑：敏感词时间区间替换为等时长屏蔽音效  | 主人声在区间内 `volume=0`，每个区间生成等长 beep 并 `adelay` 对齐到 start_time，与主人声 `amix` 输出                                       |

## 脚本位置与用法

- **路径**: `.trae/skill/审核/scripts/sensitive_audio_replace.py`
- **依赖**: 仅 Python 3 标准库 + 系统已安装的 `ffmpeg`

### 用法示例

```bash
# 在项目根或审核目录下执行
python .trae/skill/审核/scripts/sensitive_audio_replace.py \
  --sensitivity ./audit_output/sensitivity.json \
  --vocals ./audit_output/20260209-204647_vocals.mp3 \
  -o ./audit_output/20260209-204647_vocals_censored.mp3

# 指定屏蔽音（默认用 审核/asserts/dd6a0e.mp3）
python .trae/skill/审核/scripts/sensitive_audio_replace.py \
  -s ./audit_output/sensitivity.json \
  -v ./audit_output/xxx_vocals.mp3 \
  -o ./audit_output/xxx_vocals_censored.mp3 \
  --beep .trae/skill/审核/asserts/dd6a0e.mp3

# 仅打印 ffmpeg 命令不执行（调试用）
python .trae/skill/审核/scripts/sensitive_audio_replace.py -s ... -v ... -o ... --dry-run
```

### 参数说明

| 参数                   | 必填 | 说明                                                        |
| ---------------------- | ---- | ----------------------------------------------------------- |
| `--sensitivity` / `-s` | 是   | `sensitivity.json` 路径（含 `start_time`、`end_time` 毫秒） |
| `--vocals` / `-v`      | 是   | 人声音频路径（如 `*_vocals.mp3`）                           |
| `-o` / `--output`      | 是   | 输出音频路径（替换后的新音频）                              |
| `--beep` / `-b`        | 否   | 屏蔽音效文件，默认 `审核/asserts/dd6a0e.mp3`                |
| `--dry-run`            | 否   | 只打印 ffmpeg 命令，不执行                                  |

## 输入输出约定

- **输入**
  - `sensitivity.json`: 数组，每项至少包含 `start_time`、`end_time`（毫秒），与 `敏感词检测.md` 输出格式一致。
  - 人声音频: 与 av_separation 产出的人声格式一致即可（如 mp3）。
- **输出**
  - 仅生成**一条**替换后的音频文件（如 `*_vocals_censored.mp3`）。
  - 后续流程（如与 BGM 混音、字幕压制、音视频合并）由调用方或其它脚本完成，本脚本不负责。

## 实现要点

1. **等长替换**  
   每个区间 `[start_time, end_time]` 的时长 `duration_ms = end_time - start_time`。生成的 beep 片段长度严格为 `duration_ms`：
   - 若 `duration_ms > 1017`：对 `dd6a0e.mp3` 使用 `-stream_loop ceil(duration_ms/1017)-1` 再 `atrim=0:duration_sec`。
   - 若 `duration_ms <= 1017`：对 `dd6a0e.mp3` 使用 `atrim=0:duration_sec`。

2. **路径**
   - 脚本通过 `Path(__file__).resolve().parent` 定位到 `scripts/`，默认 beep 为 `parent/asserts/dd6a0e.mp3`（与文档中 `asserts/dd6a0e.mp3` 一致，注意文档里写的是 `asserts` 目录）。

3. **无敏感词**  
   若 `sensitivity.json` 为空或无有效区间，则直接 `ffmpeg -i vocals -c:a copy output`，不引入 beep。

4. **可维护性**
   - 不写死工作目录；所有路径由参数或脚本所在目录推导。
   - `BEEP_DURATION_MS = 1017` 为常量，若更换音效文件可一并修改此处。
