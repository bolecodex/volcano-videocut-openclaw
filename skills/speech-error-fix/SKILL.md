# 口播错字纠正（局部换音轨）

## 能力说明

- **输入**：带音轨的视频（如 Seedance 2.0 导出 mp4）、**要替换的时间段** `[start, end)`（秒）、**该段应读出的正确文本**（通常很短，甚至一个字）。
- **处理**：提取原音轨 → 用与项目一致的 **Ark TTS**（`synthesize_narration`）合成纠正音频 → 将补丁时长对齐到时间窗 → FFmpeg 拼接 → 原视频 **copy 视频流** + 新 AAC 音轨。
- **不做**：不自动找「哪一秒读错」——时间窗需人工听辨或用 ASR（见下「辅助定位」）。

## 前置条件

- 已安装 **FFmpeg**、**ffprobe**
- `.env`：`ARK_API_KEY`；并配置 **`ARK_TTS_ENDPOINT` 或 `ARK_TTS_MODEL`**（与 `ai_narration` 相同），否则 TTS 会退化为占位静音，无法真实纠音
- Python 依赖同 `scripts/ai_narration.py`（`openai`、`python-dotenv` 等）

## 命令行用法

在项目根目录执行：

```bash
python3 scripts/speech_error_patch.py "video/output/某成片.mp4" \
  --start 12.4 \
  --end 13.1 \
  --text "正" \
  -o "video/output/某成片_speechfix.mp4"
```

| 参数 | 说明 |
|------|------|
| `video` | 输入视频路径 |
| `--start` | 替换起点（秒，含） |
| `--end` | 替换终点（秒，不含），须大于 `start` |
| `--text` | 该窗口内应读出的**正确**文本 |
| `-o` | 输出路径；省略则生成 `*_speechfix.mp4` |
| `--voice` | TTS 音色，默认 `zh_female_huayan`（与 AI 解说一致时可尽量选原片同款音色） |
| `--speed` | TTS 语速，默认 `1.0` |

**时间窗建议**：略宽于单字时长（例如 0.4～0.9s），避免切到相邻音节；过短（&lt;0.05s）脚本会拒绝。

## 辅助定位（可选）

若不确定错音起止时间：

1. 使用技能 **asr-subtitle** / `scripts/asr_extract.py` 拉出带时间戳的台词，对照文稿找到对应片段，再填 `--start` / `--end`。
2. 或在剪辑软件 / `ffplay` 中边听边记大致秒数后微调。

## 局限与注意

- **唇形**：若画面为强口型同步，仅换音可能轻微不同步；严重依赖口型的片段更适合重新生成视频或缩短替换窗只覆盖气口。
- **音色一致**：TTS 音色与 Seedance 内置声线可能不同；尽量在火山侧选接近的 `voice`，或接受轻微音色跳变。
- **补丁长于窗口**：会截断到窗口长度，可能切掉尾音——应略加宽 `--end` 或缩短 `--text`。
- **补丁短于窗口**：末尾补静音填满窗口，听感上可能略「空」，可适当收窄时间窗。

## 与「AI 解说」的区别

| | AI 解说 (`ai_narration.py`) | 本技能 (`speech_error_patch.py`) |
|--|-----------------------------|----------------------------------|
| 范围 | 整条音轨替换 | 只替换 `[start,end)` |
| 文案 | 模型生成整段解说 | 你提供该窗内的正确短文本 |
| 场景 | 重新配音 | 错一字、少数字修正 |

## 输出

- 默认：同目录 `原名_speechfix.mp4`（H.264 视频流 copy，音轨 AAC 重编码）。
