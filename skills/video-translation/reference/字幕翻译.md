# 字幕翻译

## 流程阶段

- 阶段 5
- 上游：`ASR结果与处理.md`、`人声语速预估.md`
- 下游：`TTS音色复刻.md`、`字幕压制.md`

## 第一原则

- 禁止生成新的脚本进行内容处理。

## 输入

- `asr_split.json`
  - 来源：ASR 结果处理阶段
  - 作用：提供分句后的 `segments` 与 `words` 时间信息
- 语速信息（来自语速预估阶段）
  - 建议使用 `speed_mapping.json` / `speed_summary.json`
  - 作用：约束译文长度与节奏，尽可能对齐原始时长

## 翻译要求

1. **语言要柔和**
   - 保持自然、礼貌、不过度生硬或攻击性表达。
2. **需要本地化**
   - 语气词、称谓、习惯表达按目标语言本地习惯处理。
3. **尽可能保证时长与原始音频一致**
   - 结合 `segment.start/end` 与语速信息控制译文长度。
   - 必要时通过同义替换、短句化等方式微调节奏。
4. **人名本地化规则（剧集作用域）**
   - 在剧集目录优先读取 `mapping_name_{lang}.json`。
   - 若已有映射：必须优先复用，保持全剧一致。
   - 若不存在或缺项：补充新条目并回写到同一文件。
   - 文件示例：`mapping_name_en.json`、`mapping_name_ja.json`。

## 处理口径（基于 words）

- 翻译时必须参考 `words` 的时间分布，不只看整句文本。
- 对长句做分块翻译时，优先在 `words` 边界处切分，避免破坏时间对齐。
- 人名、专有名词优先做词级替换，避免同一角色出现多译名。

## 输出

- 主输出：`transcript_{lang}.json`
  - 例如：`transcript_en.json`、`transcript_ja.json`
  - 建议存放：`output/${剧名}/${视频名}/translates/`
- 人名映射：`mapping_name_{lang}.json`
  - 剧集作用域存放：`output/${剧名}/mapping_name_{lang}.json`
  - 每次翻译需读取并按需补充

## 质量检查建议

- 同一角色在全剧译名一致
- 译文语气柔和且本地化
- 片段时长与译文长度基本匹配（避免明显过长/过短）
