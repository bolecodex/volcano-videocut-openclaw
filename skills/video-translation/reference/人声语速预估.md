# 人声语速预估

## 流程阶段

- 阶段 4
- 上游：`人声片段剪切.md`
- 下游：`字幕翻译.md`

## 脚本说明

`count_speed_profile.py` 用于基于 Index-TTS2 复刻标准语，统计每个片段的语速（秒/字）。

同时会输出“原片段语速”口径：

- `source_speed_sec_per_char = (end - start) / 字数`
- 计数采用中英分口径：
  - 中日韩文本按“字/字符”计数（去空白、去标点）
  - 英文等空格分词语言按“词”计数（如 `hello world` 计数为 2）
- 例如中文 `你好` 计数为 2，则语速为 `时长 / 2`

本步骤直接使用 `人声片段剪切` 的输出作为输入：

- `segments/mapping.json`
- `segments/segment_*.wav`（或 `segment_*.mp3`，以 mapping 中 `file` 为准）

脚本路径：

- `scripts/base/count_speed_profile.py`

## 使用方式（uv）

在 `scripts/base` 目录下执行：

```bash
uv run count_speed_profile.py \
  --video-output-dir "/path/to/output/剧名/视频名" \
  --target-language zh
```

同一部剧多语言并发处理示例：

```bash
uv run count_speed_profile.py \
  --video-output-dir "/path/to/output/剧名/视频名" \
  --target-languages zh en ja \
  --lang-workers 3
```

> 规则：当用户输入多个目标语言时，必须对每个语言分别执行语速评估并产出对应结果（不得只评估其中一个语言）。

也可显式传入 mapping：

```bash
uv run count_speed_profile.py \
  --mapping "/path/to/output/剧名/视频名/segments/mapping.json" \
  --target-language zh
```

## 参数说明

| 参数 | 说明 | 默认值 |
| --- | --- | --- |
| `--mapping` | `mapping.json` 路径 | 无 |
| `--video-output-dir` | 视频输出目录（包含 `segments/`），可自动推断 mapping | 无 |
| `--segments-dir` | `segments` 目录路径 | `mapping.json` 所在目录 |
| `--target-language` | 目标语言（单个） | 无（与 `--target-languages` 至少传一个） |
| `--target-languages` | 目标语言列表（多个） | 无 |
| `--lang-workers` | 多语言并发数 | `1` |
| `--standard-text` | 覆盖默认标准语文本 | 按语言自动（如 `zh=你好`, `en=hello world`） |
| `--output-dir` | 语速评估输出目录 | `<视频目录>/speed_profile` |
| `--index-tts-dir` | index-tts 仓库目录 | `video-translation/scripts/vendors/index-tts` |
| `--model-dir` | Index-TTS checkpoints 目录 | `<index-tts-dir>/checkpoints` |
| `--cfg-path` | Index-TTS `config.yaml` 路径 | `<model-dir>/config.yaml` |
| `--clone-index-tts` | 缺仓库时自动克隆 | `false` |
| `--git-lfs-pull` | 克隆后执行 LFS 拉取 | `false` |
| `--use-fp16` | 启用 FP16 推理 | `false` |
| `--use-cuda-kernel` | 启用 CUDA kernel 加速 | `false` |
| `--use-deepspeed` | 启用 deepspeed | `false` |
| `--auto-install-missing-deps` | 自动安装缺失依赖 | `false` |
| `--no-skip-existing` | 已存在复刻音频时强制重跑 | `false` |
| `--limit` | 仅处理前 N 个片段（调试） | 无（全量） |

## 输出说明

默认输出到 `<视频目录>/speed_profile`，包含：

- `cloned/segment_XXXX.wav`：标准语复刻音频
- `speed_mapping.json`：每个片段语速详情
- `speed_summary.json`：语速汇总（均值/中位数/P90）

多语言模式下，输出到：

- `<视频目录>/speed_profile/<lang>/cloned/segment_XXXX.wav`
- `<视频目录>/speed_profile/<lang>/speed_mapping.json`
- `<视频目录>/speed_profile/<lang>/speed_summary.json`

其中 `speed_mapping.json` 的核心字段：

- `id`：片段 id
- `ref_file`：引用的原片段文件（来自 `segments/mapping.json`）
- `cloned_duration_sec`：复刻音频时长
- `speed_sec_per_char`：秒/字语速
- `status`：`success` / `failed`
