# 音视频合并

## 流程阶段

- 阶段 7
- 上游：`TTS音色复刻.md`、`音视频预处理.md`
- 下游：`字幕压制.md`

## 目标

将 TTS 片段按 `asr_split.json` 的时间轴放回原时间线，与背景音混合后再与静音视频合成最终视频。

## 常见问题：为什么会有“被截断感”

常见原因是片段时长超过 `end-start` 槽位后被强制裁剪，或片段头尾静音占用了可用时长，导致有效语音被压缩/截断。
方案 C 下已取消倍速与硬裁剪，优先通过去静音和借用前后间隔做软对齐。

## 合并规则

1. 每个片段的放置时间必须参考 `asr_split.json` 的 `start/end`。
2. 若片段偏长，先移除前后静音（减少“被截断感”）。
3. 若仍偏长，优先进行间隔微调：根据前后片段空隙借用可用时间窗口。
4. 若仍偏长，进行倍速微调，倍速上限由 `--max-speed` 控制（默认 `1.2`）。
5. 当前不做硬裁剪；超长片段在微调后仍可能软溢出并与后续片段叠加混音。
6. 全部片段按时间延迟后先得到配音轨，再与背景音按音量混合。
7. 混合后的整轨与静音视频 mux 生成最终视频。

## 脚本

- `scripts/base/merge_audio_video.py`
- 基于 `ffmpeg` / `ffmpeg-python`

## 输入

- `--muted-video`：静音视频（如 `muted_video.mp4`）
- `--asr-split-json`：ASR 切分结果（`asr_split.json`）
- `--tts-dir`：TTS 片段目录（如 `tts/en/`）
  - 片段命名支持：`segment_tts_0001.wav/mp3`、`segment_tts_001.wav/mp3`
- `--background-audio`：背景音路径（如 `background.mp3`）

## 输出

- `--output-video`：最终合成视频（如 `${视频名}_translate_video.mp4`）
- `--output-audio`（可选）：混合后的整轨音频（默认 `dubbed_track.wav`）

## 参数说明

| 参数 | 说明 | 默认值 |
| --- | --- | --- |
| `--muted-video` | 静音视频路径 | 必填 |
| `--asr-split-json` | `asr_split.json` 路径 | 必填 |
| `--tts-dir` | TTS 片段目录 | 必填 |
| `--background-audio` | 背景音路径 | 必填 |
| `--output-video` | 输出视频路径 | 必填 |
| `--output-audio` | 输出整轨音频路径 | 与 `output-video` 同目录 `dubbed_track.wav` |
| `--dubbed-volume` | 配音轨音量 | `1.0` |
| `--background-volume` | 背景音音量 | `0.35` |
| `--max-speed` | 超长片段倍速微调上限（限制在 `[1.0,1.2]`） | `1.2` |
| `--retry-times` | ffmpeg 阶段失败重试次数 | `3` |

## 使用示例（uv）

```bash
uv run merge_audio_video.py \
  --muted-video "/path/to/output/剧名/视频名/muted_video.mp4" \
  --asr-split-json "/path/to/output/剧名/视频名/asr_split.json" \
  --tts-dir "/path/to/output/剧名/视频名/tts/en" \
  --background-audio "/path/to/output/剧名/视频名/background.mp3" \
  --output-video "/path/to/output/剧名/视频名/视频名_translate_video.mp4"
```

## 日志与重试

- 日志前缀统一为 `[merge_av]`，输出关键阶段与统计：
  - 有效片段数、缺失片段数、去静音片段数、借间隔片段数、倍速微调片段数
- ffmpeg 阶段带重试，失败会打印尝试次数与错误信息
