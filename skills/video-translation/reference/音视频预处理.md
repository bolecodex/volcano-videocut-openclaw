# 音视频预处理

## 流程阶段

- 阶段 1
- 上游：`环境配置.md`
- 下游：`ASR结果与处理.md`

## 脚本说明

`video_asr_pipeline.py` 用于执行视频 ASR 预处理流水线，包含以下步骤：

1. 视频静音
2. 音视频分离（提取音频）
3. 人声与背景音分离
4. ASR 识别（最后一步）
5. 批量并发处理（并发上限 5，队列上限 5）

## 人声/背景音分离说明

- 该阶段会明确输出：
  - `vocals.mp3`（人声）
  - `background.mp3`（背景音）
- 这两个文件会作为后续流程输入：
  - `vocals.mp3` → `人声片段剪切.md`
  - `background.mp3` → `音视频合并.md`

脚本路径：

- `scripts/base/video_asr_pipeline.py`

## 使用 uv 执行

在 `scripts/base` 目录下使用 `uv run` 直接执行：

```bash
uv run video_asr_pipeline.py "https://example.com/video.mp4" --series-name "示例剧名"
```

## 参数说明

| 参数                   | 说明                                                                       | 默认值                   |
| ---------------------- | -------------------------------------------------------------------------- | ------------------------ |
| `input`（位置参数）    | 支持多个输入：视频文件路径、https 链接；文件夹路径最多一个（支持递归扫描） | 必填，无默认值           |
| `--output-root`        | 输出根目录（默认：`video-translation` 的父级的父级的父级 `output`）          | `../../../../../output/` |
| `--series-name`        | 剧集名（可选）                                                             | 无（不传则不按剧名分组） |
| `--max-workers`        | 并发数，最大 5                                                             | `5`                      |
| `--queue-size`         | 队列大小，最大 5                                                           | `5`                      |
| `--retry-times`        | 失败重试次数                                                               | `3`                      |
| `--retry-backoff-base` | 重试退避基数秒                                                             | `1.0`                    |
| `--asr-model`          | ASR 模型，支持 `0.6B` / `1.7B`                                             | `1.7B`                   |
| `--device`             | ASR 设备：`auto` / `cpu` / `cuda:0`                                        | `auto`                   |
| `--language`           | ASR 语言提示                                                               | 无（自动）               |
| `--log-mode`           | 日志模式：`simple` / `verbose`                                             | `simple`                 |
| `--verbose`            | 等价于 `--log-mode verbose`                                                | `false`（不传）          |

## 命名与重名处理

- 视频名称：当用户未显式指定时，优先使用视频信息生成名称（本地文件用文件名；URL 会先解码再取真实文件名）。
- 重名策略：同一批次中如果视频名称重复，按“后者覆盖前者”处理，直接删除旧输出目录后重建，不再自动追加 `_2`、`_3` 后缀。

## 输出说明

默认会在 `output/${剧名}/${视频名}` 目录下生成：

- `original_video.mp4`
- `audio.mp3`
- `muted_video.mp4`
- `vocals.mp3`
- `background.mp3`
- `asr.json`
- `status.json`

批量任务还会在剧集目录下生成：

- `pipeline_report.json`
