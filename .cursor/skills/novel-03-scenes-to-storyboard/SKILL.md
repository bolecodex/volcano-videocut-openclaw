# 场景转分镜配图

将已切分的场景文件转换为分镜数据，生成配图提示词和可视化查看器。

---

## 项目目录定位规则（必须首先执行）

> **本 skill 的所有读写操作都基于 `{项目目录}`，不是 workspace 根目录。**

`{项目目录}` 是 workspace 根目录下以剧本简称命名的子目录，由上游 skill（novel-01/02）创建：

```
{workspace根目录}/
├── {剧本简称}/          ← 这就是 {项目目录}
│   ├── style.yaml
│   ├── scenes/
│   └── ...
└── 剧本原文.md
```

### 如何定位

1. 用户指定了项目名 → 直接使用 `{workspace}/{项目名}/`
2. 用户未指定 → 列出 workspace 下的子目录，找到包含 `scenes/` 和 `style.yaml` 的目录
3. **禁止**在 workspace 根目录下直接读写产物文件

---

## 前置条件

本 skill 需要 **novel-02-script-to-scenes** skill 的输出作为输入：

```
{项目目录}/
├── {剧本名}_场景索引.yaml      # 必需
└── scenes/                      # 必需
    ├── SC_01_场景名.md
    ├── SC_02_场景名.md
    └── ...
```

---

## 输入要求

1. **场景索引文件** - `{剧本名}_场景索引.yaml`
2. **场景文件目录** - `scenes/` 下的所有场景文件
3. **`{项目目录}/style.yaml`** - 全局风格配置文件（**必需**），从中读取：
   - `style_base`：写入 `_manifest.yaml`，供下游 novel-04/06 使用
   - `image_sizes.storyboard.preset`：记录到 manifest，供 novel-04 使用
   - `video`：记录到 manifest，供 novel-06/07 使用
   - `mood_presets`：按场景类型追加色调描述到分镜提示词
   - `lighting_presets`：按时间/地点追加光照描述到分镜提示词
4. **角色资产文件**（可选）- `{剧本名}_角色资产.yaml`，包含角色ID和提示词

---

## 输出产物

```
{项目目录}/
├── {剧本名}_场景索引.yaml      # 原有（不修改）
├── index.html                   # 新增：可视化查看器
├── scenes/                      # 原有（不修改）
└── shots/                       # 新增：分镜文件目录
    ├── _manifest.yaml           # 新增：分镜索引（列出所有分镜文件）
    ├── SC_01_场景名.yaml        # 场景01的分镜数据
    ├── SC_02_场景名.yaml        # 场景02的分镜数据
    ├── ...
    ├── SC_01_场景名/            # 场景01的配图目录
    │   ├── shot_001.png
    │   └── ...
    └── SC_02_场景名/            # 场景02的配图目录
        └── ...
```

**命名规则：**
- 分镜文件：`{场景ID}_{场景名}.yaml`（与 scenes/ 下的 md 文件对应）
- 配图目录：`{场景ID}_{场景名}/`
- 配图文件：`shot_{序号}.png`

## 执行流程

### Step 1: 读取场景索引

1. 加载 `{剧本名}_场景索引.yaml`
2. 验证场景文件完整性
3. 检查是否有角色资产文件

### Step 2: 遍历场景文件

按场景索引顺序，逐个处理场景文件。

### Step 3: 单场景内分镜

在**单个场景内**划分镜头，详见 [references/shot-division-rules.md](references/shot-division-rules.md)

**核心原则：**
- ⭐ **单镜头时长：3-7 秒**（总字符数 12-28 字，超过 32 字必须拆分）
- 极短句（≤10字）应与相邻短句合并，不独占镜头
- 单条台词 >28 字 → 独占镜头，标记"上游需断句"
- 情绪转折 → 新镜头
- 人物出入场 → 考虑新镜头
- 动作高潮 → 独立镜头
- 旁白时空跳跃 → 必须新镜头
- **单镜头对话量：1-2条理想，3条仅极短句时可接受，≥4条必须拆分**

**容量预估**：`单场景镜头数 ≈ 场景总字符数 ÷ 20`

### Step 4: 生成提示词

为每个镜头生成配图提示词，详见 [references/prompt-template.md](references/prompt-template.md)

**提示词结构：**
```
[风格锚点] + [场景描述] + [角色描述@ID] + [动作/表情] + [构图/镜头]
```

**特殊类型场景处理：**
- **闪回**：提示词需包含闪回视觉标记（如暗角、褪色）
- **梦境**：提示词需包含超现实视觉元素
- **蒙太奇**：按 `estimated_shots` 预估镜头数划分快切

### Step 5: 输出分镜YAML

为每个场景生成独立的分镜文件 `shots/{场景ID}_{场景名}.yaml`，同时更新 `shots/_manifest.yaml` 索引文件。

格式详见 [references/output-format.md](references/output-format.md)

### Step 6: 运行脚本生成 HTML 查看器

```bash
python {skill_dir}/scripts/generate_storyboard.py --project "{项目目录}/"
```

脚本自动：
- 读取 `shots/_manifest.yaml` 和所有场景分镜 YAML
- 将数据嵌入 HTML 模板
- 生成自包含的 `{项目目录}/index.html`（无需 HTTP 服务器，双击即可打开）

```bash
open "{项目目录}/index.html"
```

### Step 7: 分镜自检

完成划分后，检查：

```
⭐ 时长检查（最高优先级 —— 目标 3-7 秒/镜头）：
- [ ] 每个镜头总字符数 ≤ 28？（目标 3-7s）
- [ ] 无镜头总字符数 > 32？
- [ ] 单条台词 > 28字的已独占镜头并标记"上游需断句"？
- [ ] 极短句（≤10字）已与相邻句合并、不独占镜头？
- [ ] 镜头间时长差异不超过 2 倍？
- [ ] 无台词在多个镜头间重复？

条数检查：
- [ ] 所有镜头对话条数 ≤ 2？（理想）
- [ ] 无镜头对话条数 ≥ 4？

时空一致性检查：
- [ ] 每个镜头内的旁白描述同一时空？
- [ ] 旁白跨时空（如"父亲死后...母亲也..."）已拆分为多镜头？
- [ ] 镜头的 scene_ref 正确引用了场景ID？
- [ ] 特殊类型场景的镜头继承了视觉标记？

整体检查：
- [ ] 单场景镜头数 ≈ 场景总字符数 ÷ 20？
- [ ] 蒙太奇场景的镜头数符合预估？
- [ ] 每个镜头能用一张图清晰表达？
```

## 镜头类型

| 镜头 | 用途 | 适用场景 |
|------|------|----------|
| 特写 | 情绪、细节 | 独白、内心戏 |
| 中景 | 对话、互动 | 两人交流 |
| 全景 | 场景建立 | 开场、转场 |
| 远景 | 氛围、史诗感 | 环境展示 |

---

## 常见场景类型的视觉处理

| 场景类型 | 色调 | 光线 | 典型元素 |
|----------|------|------|----------|
| 室内日景 | 暖调 | 侧光 | 家具、窗户光斑 |
| 室内夜景 | 暖黄 | 油灯/烛光 | 暗角、光晕 |
| 室外日景 | 明亮 | 自然光 | 天空、远景 |
| 室外夜景 | 冷调 | 月光/火把 | 阴影、星空 |
| 梦境/幻想 | 饱和 | 自发光 | 雾气、超现实 |
| 闪回 | 褪色 | 柔光 | 暗角、颗粒感 |

## HTML 查看器

### 生成方式

由 `scripts/generate_storyboard.py` 读取 YAML 数据后生成自包含 HTML：

```bash
# 生成（数据内嵌，无需服务器）
python {skill_dir}/scripts/generate_storyboard.py --project "{项目目录}/"

# 打开
open "{项目目录}/index.html"
```

### 脚本参数

| 参数 | 缩写 | 必填 | 说明 |
|------|------|------|------|
| `--project` | `-p` | 是 | 项目目录（包含 `shots/` 子目录） |
| `--template` | `-t` | 否 | HTML 模板路径（默认使用 skill 内置模板） |
| `--output` | `-o` | 否 | 输出路径（默认: `{项目目录}/index.html`） |

### 功能特性

- 搜索台词、角色、场景
- 按图片状态过滤（待生图/已生图）
- 点击图片全屏查看
- 一键复制配图提示词
- 场景分隔标题（按场景分组展示）
- 台词内嵌音频播放器（如有 `audio_url`）
- 键盘快捷键：`/` 搜索，`Esc` 关闭弹窗

---

## 注意事项

1. **只在单场景内分镜** - 不跨场景操作，场景边界由 novel-02-script-to-scenes 决定
2. **继承场景类型** - 镜头的视觉风格继承自所属场景
3. **角色一致性** - 提示词必须引用角色ID（如 `@szj_chentu`）
4. **场景连续性** - 同一场景的多个镜头保持相同的场景描述
