# 场景切分规则

## 核心原则

场景的本质是**时空叙事单元**。每个场景应该在**同一地点、同一时间段、同一叙事类型**内完成。

> **关键理念**：场景是按「时空」划分，分镜是按「画面」划分。
> 先把时空边界切清楚，后续分镜才能均匀分布。

---

## 场景类型定义

| 类型 | 代码 | 说明 | 视觉特征 |
|------|------|------|----------|
| 现实 | reality | 主时间线的当前事件 | 正常色调 |
| 闪回 | flashback | 过去发生的事件 | 可加滤镜/暗角 |
| 梦境 | dream | 梦中/幻想场景 | 超现实色调 |
| 蒙太奇 | montage | 时间压缩的快切段落 | 多画面并置 |
| 预告 | prologue | 开篇悬念/预告 | 特殊处理 |

---

## 强制切分条件

以下情况**必须**划分新场景：

### 1. 地点变化

当场景发生物理空间转移时：

```
触发词：
- "来到..."、"走进..."、"回到..."
- "在xxx（地点）"
- "铺子里"、"房间内"、"街上"、"院落"、"书房"
- 任何明确的地点转移

示例：
旁白：我们的"洞房"，就在猪肉铺后面。  ← 新场景：从外部转入卧房
旁白：穿过街巷，最终停在了一处僻静院落前。  ← 新场景：从街巷到院落
```

### 2. 时间跳跃

当时间发生明显跳跃时：

```
触发词：
- "第二天"、"三天后"、"一年后"、"那晚"
- "天亮"、"天色泛白"、"夜里"、"子时"
- "午后"、"黄昏"、"早饭时"、"晚饭时"
- 任何时间状语

示例：
旁白：直到那晚，瓢泼大雨。  ← 新场景：时间跳跃到特定夜晚
旁白：三天后的黄昏，京兆府的衙役突然封锁了街口  ← 新场景：时间跳跃
```

### 3. 叙事类型切换

当叙事类型发生变化时：

```
现实 → 闪回：
- "想当初"、"那日"、"当年"、"回想起"
- "苏家被抄那日"

现实 → 梦境：
- "眼前的xx消失了"、"坠入"、"恍惚间"
- 描述超现实场景的开始

闪回/梦境 → 现实：
- 需要识别退出点
- 通常伴随"醒来"、"回过神"、"抽回手"等

蒙太奇进入：
- "日子一天天过去"、"清洗开始了"
- 时间压缩的概述性描述
```

### 4. 同地点不同时段

**默认作为新场景处理**，这是避免"分镜不均匀"的关键：

```
示例（原本可能被当作一个场景）：
旁白：直到那晚，瓢泼大雨。（夜晚）
...
旁白：我缩在床角...直到天色泛白。（时间跳跃）
旁白：雨停了。陈屠醒来...（天亮）
旁白：早饭时，我碗里多了个剥好的水煮蛋。（早饭）

正确处理：
- SC_07: 雨夜惊梦（夜晚 → 天色泛白前）
- SC_08: 雨后清晨（天亮 → 早饭）
```

## 不切分的情况

以下情况**不应**划分新场景：

### 1. 连续对话

同一地点、同一时段内的连续对话：

```
街坊甲：苏家的大小姐，金枝玉叶...
街坊乙：嘘——家道中落，配个屠户...
← 同一场景内的连续对话，不切
```

### 2. 同场景细节描写

同一时空内的细节旁白：

```
旁白：他剁完，会一声不吭地走到隔壁的烧饼铺...
旁白：买一个刚出炉的芝麻烧饼回来，塞我手里。
陈屠：吃。
← 同一场景内的连续动作，不切
```

### 3. 情绪推进

同一时空内的情绪连续变化：

```
旁白：我心头一跳。
旁白：京兆府尹手下有个酷吏，就叫李三...
旁白：京城里叫李三的没有一万也有八千...
← 同一场景内的心理活动，不切
```

## 闪回/梦境的成对识别

闪回和梦境必须**成对识别**，确保有明确的进入点和退出点。

### 闪回识别

```
进入点标志：
- 明确的时间回溯词："那日"、"当年"、"想当初"
- 叙事视角切换到过去

退出点标志：
- 回到当前时间线
- 叙事视角切换回现在
- 注意：退出点可能没有显式标记，需要根据内容判断

示例：
L47: 旁白：想当初，我也是京城路人人称羡的苏家千金苏晚。  ← 闪回触发
L49-84: 闪回内容（苏府抄家、人市被买、新婚之夜）
L86: 旁白：日子一天天过去。  ← 闪回结束，回到现在
```

### 梦境识别

```
进入点标志：
- "眼前的xx消失了"
- "坠入"、"进入"异空间描述
- 超现实场景开始

退出点标志：
- "猛地抽回手"、"惊醒"
- 回到现实环境描述
- "油灯还在桌上摇曳"等现实锚点

示例：
L171: 旁白：就在我触碰到他皮肤的一刹那——  ← 梦境进入前
L172: 旁白：眼前的卧房消失了...  ← 梦境开始
L185: 旁白：我猛地抽回手！  ← 梦境结束
L186: 旁白：眼前的血雾世界骤然崩塌。  ← 确认退出
```

## 蒙太奇场景处理

蒙太奇场景的台词数与镜头数**不成正比**，需要特殊处理。

### 识别蒙太奇

```
触发词：
- "日子一天天过去"
- "清洗，开始了"
- "短短一个月内"
- 时间压缩的概述性描述

特征：
- 跨越较长时间段
- 多个并行事件概述
- 没有具体的单一场景焦点
```

### 蒙太奇场景元信息

必须在场景元信息中标注：

```yaml
- id: SC_24
  name: 清洗蒙太奇
  type: montage
  estimated_shots: 6-8  # 预估快切镜头数
  time_span: 一个月     # 时间跨度
  notes: 多个暴毙事件并置，需要快切节奏
```

## 场景容量标准

### 台词数参考

| 等级 | 台词数 | 处理方式 |
|------|--------|----------|
| ✅ 理想 | 15-30条 | 直接使用 |
| ⚠️ 偏小 | <15条 | 考虑是否可合并（但不强制） |
| ⚠️ 偏大 | 31-50条 | 复核是否漏切边界 |
| ❌ 过大 | >50条 | 必须复核，很可能漏切了时间跳跃 |

### 过大场景的常见原因

1. **漏切时间跳跃**：同地点但跨越了多个时段
2. **漏切情节单元**：包含多个独立的情节段落
3. **闪回/梦境未独立**：闪回或梦境混在现实场景中

### 过大场景的处理建议

```
示例：SC_19 坦白盟约（41条台词）

内容分析：
- 陈屠照顾苏晚（317-319）
- 苏晚坦白"我知道"（320-334）
- 陈屠倾诉过去（335-351）
- 夫妻立约（353-357）

建议：
- 可考虑按情绪段落再拆分
- 或保留为一个场景，在分镜阶段拆成4个镜头组
- 选择取决于是否需要在场景层面体现结构
```

## 场景自检清单

完成切分后，逐项检查：

### 边界检查（硬性）
- [ ] 所有明确时间跳跃处都落在场景边界上？
- [ ] 所有地点转换处都落在场景边界上？
- [ ] 闪回/梦境都成对识别（有进入点和退出点）？
- [ ] 叙事类型切换处都有场景边界？

### 容量检查
- [ ] 无场景台词数 > 50？
- [ ] 各场景台词量差异不超过5倍？（最大/最小）
- [ ] 总场景数 ≈ 总台词数 ÷ 25？

### 类型检查
- [ ] 蒙太奇场景已标注"预估快切镜头数"？
- [ ] 闪回场景已标注时间线信息（过去的哪个时间点）？
- [ ] 梦境场景已标注视觉风格提示？

### 一致性检查
- [ ] 场景命名清晰，能体现核心内容？
- [ ] 场景ID连续，无跳号？
- [ ] 所有台词都被分配到场景中，无遗漏？
