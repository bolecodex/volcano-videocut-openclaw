# 角色提取规则详解

## 1. 三行结构规范

每个角色必须严格输出三行：

```
- **风格：**{必须与STYLE_BASE一致}
- **形象：**{角色名} (@{角色ID}) {年龄/发型/样貌/服装/动作气质}
- **视图：**{白底全身设定图 / 白底Q版全身设定图 / 白底妖形全身设定图}
```

## 2. 形象描述顺序

严格按以下顺序描述：

```
{角色名} (@{角色ID}) {年龄}{性别}，{发型发色}，{面部特征}，{体型}，{服装}，{配饰}，{气质}
```

### 2.1 年龄性别
- 男性：少年(12-15)/青年(18-25)/中年(35-50)/老年(60+)
- 女性：少女(12-15)/青年(18-25)/中年(35-50)/老年(60+)
- 示例：`20岁男性`、`18岁女性`、`中年男性`

## 3. 不可变特征规范

### 3.1 必须包含的特征
1. 发型发色（最重要的识别特征）
2. 标志性服装主色
3. 标志性配饰/道具
4. 面部最显著特征
5. 体型特征

### 3.2 不可变特征示例

**主角示例：**
```
### 不可变特征
1. 黑发束冠，发丝根根分明
2. 剑眉星目，眼神清冷
3. 白色仙袍，金边装饰
4. 腰佩青锋剑，剑穗白色
5. 身形修长，肩宽腰窄
6. 气质清冷出尘
```

**配角示例：**
```
### 不可变特征
1. 黑发如瀑及腰
2. 柳眉杏眼，眼神温柔
3. 淡紫色襦裙，绣花装饰
4. 手持白玉笛
5. 身材纤细，气质温婉
```

## 4. 角色ID命名规范

### 4.1 格式
```
@{项目缩写}_{角色拼音或英文}
```

### 4.2 项目缩写建议
| 项目类型 | 缩写示例 |
|----------|----------|
| 剑来 | jl |
| 都市修仙 | dsxx |
| 霸道总裁 | bdcz |
| 末日生存 | mrsc |

### 4.3 命名规则
- 全小写
- 下划线分隔
- 拼音首字母或完整拼音
- 同名加数字后缀

### 4.4 示例
```
@jl_chenfeng      # 剑来-陈风
@jl_ningyu        # 剑来-宁雨
@jl_huwei_jia     # 剑来-护卫甲
@jl_huwei_yi      # 剑来-护卫乙
@dsxx_zhangsan    # 都市修仙-张三
```

## 5. 特殊角色处理

### 5.1 系统/AI角色
```
## 系统 (@jl_system)

- **风格：**真人写实高清，超细节刻画，光影细腻，青蓝科技光效
- **形象：**系统 (@jl_system) 无实体，以光幕/悬浮文字/语音形式出现
- **视图：**无

### 表现形式
- 光幕：青蓝色半透明面板，科技感边框
- 文字：发光字体，打字机效果
- 语音：电子合成音，无画面主体
```

### 5.2 妖兽/非人类
```
## 血狼 (@jl_xuelang)

- **风格：**真人写实高清，超细节刻画，光影细腻，暗红妖气色调
- **形象：**血狼 (@jl_xuelang) 巨型妖狼，通体漆黑，双眼泛红光，利爪如刀
- **视图：**白底妖形全身设定图

### 不可变特征
1. 通体漆黑皮毛
2. 双眼泛红光
3. 利爪如刀
4. 獠牙外露
5. 黑雾缭绕
```

### 5.3 变身角色
```
## 陈风（魔化） (@jl_chenfeng_mohua)

- **风格：**真人写实高清，超细节刻画，古风人像写实，光影细腻，暗红邪气色调
- **形象：**陈风魔化 (@jl_chenfeng_mohua) 与正常形态相同身形，黑发变暗红，瞳孔变血红，周身黑气缭绕

### 与正常形态差异
- 发色：黑发 → 暗红
- 瞳色：黑色 → 血红
- 气质：清冷 → 狂暴
- 特效：无 → 黑气缭绕
```

## 6. 角色提示词生成规则

### 6.1 角色立绘模板（白底设定图）

> **关键规则**：角色立绘 prompt 必须使用 `style_base_character`（角色专用风格词），
> **禁止**使用 `style_base` 中的氛围/色调/光线/背景词（如"氛围感拉满""宫廷华丽色调""暖黄宫灯光线""背景虚化"等），
> 这些词会严重干扰白底效果，诱导模型生成场景背景。

```
{1girl/1boy}, solo, only one person in the image, {STYLE_BASE_CHARACTER}，{年龄}{性别}，{发型发色}，{面部特征}，{体型}，{服装}，{配饰}，{气质}，正面站立，面对镜头，全身像，双手自然下垂，solid pure white background，nothing else in the background，ultra detailed
```

#### 模板要素说明

| 位置 | 内容 | 说明 |
|------|------|------|
| **前缀** | `{1girl/1boy}, solo, only one person in the image` | **必须**放在最前面，`1girl`/`1boy` 按性别选用，三重约束防止多人/多视角 |
| **风格** | `{STYLE_BASE_CHARACTER}` | 只含画质词，不含氛围/场景词 |
| **角色** | 年龄→发型→面部→体型→服装→配饰→气质 | 按顺序描述 |
| **姿势** | `正面站立，面对镜头，全身像` | 固定姿势 |
| **背景** | `solid pure white background，no background elements` | **双重强化**白底 |
| **后缀** | `ultra detailed` | 高清细节 |

> **禁止使用** `character design reference sheet` / `character sheet` / `turnaround` 等词，
> 这些词会触发多视角（正面+侧面+背面）布局，导致图中出现多个人。

#### STYLE_BASE_CHARACTER 生成规则

从 `style.yaml` 读取 `style_base_character` 字段。若无此字段，则从 `style_base` 中**自动剥离**以下类型的词：

**必须剥离的词（会诱导背景）：**
- 氛围词：`氛围感拉满`、`氛围感强`
- 色调词：`宫廷华丽色调`、`青绿山水色调`、`仙侠清冷氛围`、`暖黄色调`
- 光线词：`暖黄宫灯光线`、`暖黄色的光线`、`烛光氛围`
- 背景词：`背景虚化氛围感强`、`背景虚化`
- 动态词：`动态自然`

**保留的词（纯画质词）：**
- `真人写实高清`、`超细节刻画`、`古风人像写实`
- `光影细腻`、`服饰纹理清晰`、`面部神态精准`

### 6.2 多角度模板
```
solo, one single person only, {STYLE_BASE_CHARACTER}，{角色完整描述}，{角度描述}，solid pure white background，no background elements，ultra detailed
```

角度描述：
- 正面特写：正面特写，面部清晰，胸部以上
- 左侧45度：左侧45度角，半侧面
- 右侧45度：右侧45度角，半侧面
- 左侧面：左侧面90度，完整侧脸
- 右侧面：右侧面90度，完整侧脸
- 背面：背面视角，展示发型和背部
- 俯视：俯视角度，从上往下看
- 仰视：仰视角度，从下往上看

## 7. 质量检查清单

- [ ] 是否所有角色都有唯一ID？
- [ ] 三行结构是否完整（风格/形象/视图）？
- [ ] 形象描述是否按顺序（年龄性别/发型/面部/体型/服装/气质）？
- [ ] 不可变特征是否≥3条且≤7条？
- [ ] 不可变特征是否足够具体可视化？
- [ ] 风格是否引用STYLE_BASE？
- [ ] 角色提示词是否可直接用于生图？
- [ ] 特殊角色（系统/妖兽）是否有正确的视图说明？
