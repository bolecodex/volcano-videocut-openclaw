# 剧本 → 对话脚本 + 场景切分（一步完成）

直接从剧本原文，**一次推理**同时完成：说话人识别、音色映射、台词断句、场景边界切分。

> **为什么合并？** 原文包含最丰富的上下文信号（"XX道"、段落空行、"第N集"、地点描述），拆成两步后中间产物会丢失这些线索，导致说话人推断和场景边界识别都变差。一步完成可避免错误传播，速度翻倍。

---

## 项目目录确定规则（必须首先执行）

> **所有产物都必须输出到项目子目录，不是 workspace 根目录。**

每个剧本拥有独立的项目子目录，位于 workspace 根目录下：

```
{workspace根目录}/
├── {剧本简称}/          ← 这就是 {项目目录}
│   ├── style.yaml
│   ├── {剧本简称}_对话脚本.md
│   ├── {剧本简称}_场景索引.yaml
│   └── scenes/
├── 另一个剧本/
└── 剧本原文.md          ← 剧本原文可以在根目录
```

### 如何确定 `{剧本简称}`

从剧本标题中提取简短名称（2-8 字），去掉副标题和标点，作为目录名和文件前缀。

| 剧本标题 | 剧本简称 | 项目目录 |
|----------|---------|---------|
| 全班穿成NPC，我把宫斗玩成总动员 | 全班穿成NPC | `{workspace}/全班穿成NPC/` |
| 杀猪匠夫君 | 杀猪匠夫君 | `{workspace}/杀猪匠夫君/` |

### 执行前必须检查

1. **检查 `{项目目录}` 是否存在** → 不存在则 `mkdir -p` 创建
2. **检查 `{项目目录}/style.yaml` 是否存在** → 不存在则创建（参考已有项目或根据剧本风格新建）
3. **所有输出产物**都**必须写入 `{项目目录}/` 下**，禁止写在 workspace 根目录
4. 若用户已有同名目录（已执行过 novel-01），直接复用该目录

## 输出产物

```
{项目目录}/
├── {剧本名}_对话脚本.md         # 音色映射表 + 全量格式化台词（flat list）
├── {剧本名}_场景索引.yaml       # 场景结构数据
└── scenes/                      # 独立场景文件
    ├── SC_01_场景名.md
    ├── SC_02_场景名.md
    └── ...
```

**产物关系**：`对话脚本.md` = 音色映射表 + 所有 `scenes/*.md` 中对话内容的顺序拼接。

## 执行流程

### Step 1: 通读原文，提取全局信息

1. **识别所有人物**：从对话标注（"XX说/道/问"）和叙述中提取
2. **分配音色**：为每个人物生成音色描述（详见 [format-rules.md](references/format-rules.md) §3）
3. **识别叙述视角**：第一人称（"我"= 主角）/ 第三人称
4. **统计全文规模**：预估场景数 ≈ 原文段落数 ÷ 10（或对话条数 ÷ 25）

### Step 2: 逐段处理——同时做对话格式化 + 场景边界标记

从头到尾扫描原文，对每一段落：

**A. 对话格式化**（原 Step 02 的工作）：
- 叙述性文字 → `旁白：`
- 引号内对话 → `人物：`（根据"XX道"、上下文推断说话人）
- 去引号、保留标点、过滤章节序号
- **⭐ 5-15字断句**：超过15字在逗号/句号处拆分，每段不少于5字
- 详见 [format-rules.md](references/format-rules.md)

**B. 场景边界识别**（原 Step 03 的工作）：
- 遇到**地点变化**（"来到/走进/回到/在XX"）→ 标记新场景
- 遇到**时间跳跃**（"第二天/那晚/天亮/三天后/一年后"）→ 标记新场景
- 遇到**叙事类型切换**（闪回/梦境/蒙太奇的进出）→ 标记新场景
- 遇到**同地点不同时段** → 默认新场景
- 章节标记（"第N集"）的位置本身就是强场景边界
- 详见 [scene-division-rules.md](references/scene-division-rules.md)

### Step 3: 输出三个产物

1. **场景文件** `scenes/SC_XX_场景名.md`：每个场景包含元信息表 + 该场景的对话内容
2. **场景索引** `{剧本名}_场景索引.yaml`：全部场景的结构数据
3. **对话脚本** `{剧本名}_对话脚本.md`：音色映射表 + 所有场景对话的顺序拼接

格式详见 [output-format.md](references/output-format.md)

### Step 4: 自检

```
对话质量：
- [ ] 所有对话正确识别了说话者？
- [ ] 无单条台词超过 15 个中文字符？
- [ ] 无碎片行（<5字且非完整短句）？
- [ ] 每行只有一个说话人？

场景边界：
- [ ] 所有时间跳跃处都落在场景边界上？
- [ ] 所有地点变化处都落在场景边界上？
- [ ] 闪回/梦境成对识别？
- [ ] 无场景台词数 > 50？
```

## 超长剧本处理策略

当剧本超过 500 行时，按以下策略分批处理：

1. **先通读全文**，提取人物列表 + 音色映射 + 章节/集数结构
2. **按章节分批**：每批处理 2-3 集（或 200-300 行原文）
3. **每批输出**该段的场景文件，场景 ID 全局连续
4. **最后合并**：拼接场景索引 YAML，拼接对话脚本

## 场景类型

| 类型 | 代码 | 说明 | 视觉特征 |
|------|------|------|----------|
| 现实 | reality | 主时间线的当前事件 | 正常色调 |
| 闪回 | flashback | 过去发生的事件 | 可加滤镜/暗角 |
| 梦境 | dream | 梦中/幻想场景 | 超现实色调 |
| 蒙太奇 | montage | 时间压缩的快切段落 | 多画面并置 |
| 预告 | prologue | 开篇悬念/预告 | 特殊处理 |

---

## 常见边界触发词

| 类型 | 触发词示例 |
|------|-----------|
| 地点变化 | 来到、走进、回到、在xx、铺子里、院落、书房、殿内、宫里 |
| 时间跳跃 | 第二天、那晚、天亮、午后、黄昏、三天后、一年后、晚上、中秋宴当晚 |
| 闪回进入 | 想当初、那日、当年、回想起 |
| 梦境进入 | 眼前消失了、坠入、梦中、恍惚间 |
| 蒙太奇 | 日子一天天过去、不到两年时间、短短一个月 |
| 章节边界 | 第N集、纯数字行（作为场景分界参考，但过滤不输出） |
